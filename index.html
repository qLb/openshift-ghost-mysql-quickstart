    This is a fork of the original ghost quickstart available on openshift. It is a basic quickstart to get Ghost running on OpenShift. It uses MySQL as a database instead of SQLite3. It has been updated from Ghost 0.5 to 0.5.7.

    To install it from the command line make sure you have the latest rhc gem installed and then run the following command: replacing 'ghost' if you want your app to be named something else

    rhc app create ghost nodejs-0.10 mysql-5.5 --env NODE_ENV=production --from-code https://github.com/lemur3/openshift-ghost-mysql-quickstart.git

    'ghost' will be the name of your application in the above example.

    Note these OpenShift specific changes carried over from the original quickstart:

    1. The content/data and content/images directories have been removed. They are created on the server and symlinked to your $OPENSHIFT_DATA_DIR so that posts and uploaded images will persist across 'git pushes'
    2. Even though the node.js cartridge itself is scalable, this application will not play nice with scaling right now because the images are stored on disk, and since OpenShift does not currently support shared physical disk storage across scaled gears, this cartridge will not scale.
    3. Upon completion of the install you'll want to use a secure connection to https://yourappname-yourdomain.rhcloud.com/ghost/ in order to set up your administrator account. You may get a blank page if you connect to http://.
    4. If you use a custom domain, modify the "url:" field in the 'production' section of the config.js file, it should look something like: url: 'http://www.example.com', (with the comma at the end)
